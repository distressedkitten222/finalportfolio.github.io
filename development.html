Music Album Selector: React Development

Overview

Objective: To explore and become familiar with React! To create an interactive web application with a focus on using components and internal data state.

Role: Designer, Developer

Concept

A Filterable, Sortable Album List

There are so many factors to consider when choosing what music to listen to. Obviously, who’s singing, but less obviously:

  • Do I need to be able to understand the words, or will a language I know distract me from my UIUX homework? What language?
  • Do I want to be transported to spring break 2019, or the angsty phase of 2020 COVID lockdown? What year is the album from?
  • Will I be able to listen to the whole album in one car ride while looking out of the window like I’m in a music video? How many songs is in it?

With questions such as these to tackle, I thought it would be nice to be able to easily filter and sort music options to discover what to listen to, instead of trying to work it out in your head or 'stumble' upon something in a playlist.


My application displays music albums, with multiple filter and sort options for easy selection. You can Favourite and Unfavourite albums, and the Favourites section will aggregate the total minutes and total number of songs of all your Favourites! (I chose to display these because it helps in making playlists of a certain length without having to repeatedly add and remove songs/albums to see how the playlist length.)

Usability Considerations

To increase efficiency, learnability, and memorability, my application has:

  • A clear, well-defined layout: information is well-organized, so it’s clear to users what is what and easy to find what you need.
  • Intentional use of whitespace to increase readability.
  • Intuitive UI elements like checkboxes and radio buttons for filtering and sorting respectively: these are well-known, so users can quickly understand how things work.
  • Consistency in UI elements, font, and general design.
  • Clear hierarchy: contrasting sizes, styles, colours, and weights of text make the information hierarchy easy to distinguish.
  • Simple design and user-friendly colour scheme: the user isn’t distracted from content by loud colors or slowed down by lack of readability or difficult font.

In general, these considerations make the functions of the application easy to understand, and the application intuitive to use.

Learning

The motivation behind this project was learning! Here are some things I learned:

  • React components are so efficient! They save tons of time and effort, because you can reuse them anywhere instead of recoding the same thing multiple times, and one change in a component updates it everywhere it's used. They also made my code much easier to read.
  • The different ways in which state and props can be used to control an application, and how to dynamically change state.
  • Before building this, I didn't think about the different options for filter combinations (both must match, one or the other, etc.). It was interesting to play around and figure out what made sense as a user!

Next Steps

While I learned a lot, I'm not done yet. Moving forward, I'd like to:

  • Add more filter categories and try out different combinations of them.
  • Try implementing sort in different ways - for example, a dropdown menu.
  • Add way more music! For now, the application displays my favourite albums :) but perhaps I can try using a music API!
